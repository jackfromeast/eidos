大多数人过度思考组织。他们创建复杂的文件夹层次结构和复杂的标签系统，然后花在组织上的时间比实际使用的时间还多。

Eidos 中的空间工作方式不同。把空间想象成为您的数据拥有一个完全独立的宇宙。大多数时候，您希望所有东西都在一个宇宙中——您的笔记、您的项目、您的随机想法都相互连接并可以一起搜索。

但有时分离是有意义的。也许您想要将工作和个人事务完全分开。或者您正在处理一个包含敏感数据的项目，不应该与其他所有内容混合。这时您就创建多个空间。

关键洞察是：每个空间都有自己的数据库。不仅仅是自己的文件夹，而是自己完全隔离的数据存储。这就像拥有独立的计算机，只是它们恰好在同一台机器上运行。

## 在磁盘上的样子

<FileTree>

- my-eidos-data
  - spaces
    - work
      - db.sqlite3 ← 这里存放所有工作数据
      - files
        - some-presentation.pdf
        - meeting-notes-attachment.png
    - personal
      - db.sqlite3 ← 完全独立的数据库
      - files
        - vacation-photos.jpg
        - recipe-ingredients.pdf
    - secret-project
      - db.sqlite3 ← 再次，完全隔离
      - files
        - prototype-designs.sketch

</FileTree>

优雅之处在于简单性。没有复杂的权限，没有带有复杂访问控制的共享数据库。当您需要时干净地分离，当您不需要时将所有内容放在一起。
