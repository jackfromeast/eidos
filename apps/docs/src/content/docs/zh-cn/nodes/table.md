表格是 Eidos 认真对待数据的地方。虽然文档非常适合思考和写作，但表格适用于当您需要结构、关系以及以不同方式切片和切块信息的能力时。

## 重要的架构决策

以下是可能让您惊讶的事情：大多数"类似数据库"的生产力工具实际上并没有在底层使用真正的数据库。它们通常将所有内容存储在一个巨大的表中，并使用巧妙的技巧使其看起来像您有多个表。

我们做了相反的事情。您在 Eidos 中创建的每个表都成为一个实际的数据库表。当您创建一个"项目"表时，您会在 SQLite 中获得一个名为 `tb_<node_id>` 的真实数据库表。

为什么这很重要？性能和完整性。真正的数据库表具有真正的索引、真正的外键和真正的查询优化。您的数据表现得像数据，而不是数据的模拟。

## 每个表都获得的内容

每个表都以两个基本字段开始：

- `_id` - 每行的唯一标识符
- `title` - 因为人类需要为事物命名

这些不仅仅是便利功能。`_id` 字段是使表之间的关系成为可能的因素。`title` 字段是当您在表之间链接或在不同上下文中查看数据时使您的数据可读的因素。

## 重要的字段类型

### 基本字段

这些是您的基础：

- **文本** - 用于单词和句子
- **数字** - 用于数量和计算
- **日期** - 用于基于时间的数据
- **复选框** - 用于是/否决策
- **选择** - 用于从预定义列表中选择
- **多选** - 用于标签和类别
- **文件** - 用于附件和媒体

### 计算字段

这些是表格变得有趣的地方：

- **公式** - 就像 Excel 公式，但用于您的结构化数据
- **链接/关系** - 连接不同表中的行
- **查找** - 从相关表中提取数据

:::tip

Eidos 的公式系统建立在 SQLite 的 generated column 特性之上。这意味着什么？您使用标准的 SQL 语句来定义公式，而不需要学习另一种蹩脚的方言。这是真正的 SQL，具有真正的数据库性能。

为什么这很重要？当您需要复杂计算时，您拥有 SQL 的全部功能。需要聚合？窗口函数？子查询？没问题。

对于更高级的用例，Eidos 还提供了 UDF（用户定义函数）扩展。您可以使用 JavaScript 来实现复杂的业务逻辑，然后在您的 SQL 公式中调用这些函数。这给了您两全其美的结果：SQL 的性能和 JavaScript 的灵活性。

:::

### 系统字段

这些会自动跟踪自己：

- **created_at** - 行的创建时间
- **last_edited_at** - 最后一次被修改的时间
- **created_by** - 谁创建了它
- **last_edited_by** - 谁最后修改了它

## 查看数据的三种方式

表格支持三种视图类型，每种都针对不同的思考方式进行了优化：

**网格视图**是您的默认选择。它看起来像 Excel 并且像 Excel 一样工作。非常适合直接编辑数据、排序、过滤和获得全面概览。

**画廊视图**适用于您的数据包含图像或当您想要一次看到每个项目的更多信息时。想想 Pinterest，但用于您的结构化数据。

**看板视图**将您的表格转换为项目板。选择一个选择字段进行分组，突然您就有了带有可拖拽卡片的列。这是相同的数据，只是组织方式匹配您对工作流程的思考方式。

优雅之处在于这些只是同一信息的不同镜头。改变视图，您就改变了与数据交互的方式，但底层结构保持不变。
